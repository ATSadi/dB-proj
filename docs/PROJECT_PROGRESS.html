Project Overview

A database-driven system for managing campus maintenance complaints. Students report issues by location; supervisors assign workers; the Oracle database enforces SLA deadlines, logs status changes, and supports analytics through PL/SQL triggers, procedures, functions, and views. A basic web interface connects to Oracle via Node.js.

PL/SQL Components Implemented

  • 7 Triggers — SLA deadlines, status audit log, worker availability, performance scoring, chronic issue detection
  • 6 Procedures — assign worker, monthly report, SLA escalation, safe transactions
  • 4 Functions — worker performance, resolution time, student summary, SLA calculator
  • 5 Views — active complaints, overdue, worker performance, monthly summary, chronic issues
  • Seed data — 14 users, 24 locations, 38 complaints
