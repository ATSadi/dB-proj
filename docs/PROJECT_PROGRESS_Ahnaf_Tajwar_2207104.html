1. Project Overview

A campus maintenance complaint management system built on Oracle Database (PL/SQL) with a basic Node.js + HTML/CSS/JS web interface. Students submit maintenance complaints by location and category; supervisors assign specialized workers; the database enforces SLA deadlines, logs status changes, detects chronic issues, and tracks worker performance.

2. Work Completed So Far

  • Database design — 9 normalized tables (3NF), ERD, data dictionary, constraints, sequences
  • Seed data — 14 users, 24 locations, 38 sample complaints with assignments and feedback
  • PL/SQL layer — 7 triggers, 6 stored procedures, 4 functions, 5 analytical views
  • Advanced SQL — SLA analysis, worker overload, monthly trends, chronic issue queries
  • Backend API — Express server connected to Oracle via oracledb
  • Basic frontend — Role-based login and separate portals for student, worker, admin/supervisor
  • Documentation — Schema design, Oracle setup guide, demo SQL scripts
Current UI is functional but basic. Core focus so far has been database logic and end-to-end connectivity. Visual design and advanced features are planned next.

6. Supervisor vs Admin

Both roles share the same dashboard UI in the current version. In the database, supervisor can assign workers, monitor complaints, and run escalation. Admin has broader access including full table control and all reports. Demo logins: omar.s@campus.edu (supervisor), admin@campus.edu (admin).
